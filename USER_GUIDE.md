## 1. 시작하기

### 회원가입

1. 우측 상단의 **회원가입** 메뉴를 클릭합니다.
2. 사용자명과 비밀번호를 입력합니다.
3. **비밀번호 규칙**을 반드시 지켜주세요.
   - 최소 8자 이상
   - 영문, 숫자, 특수문자 포함
4. 가입이 완료되면 로그인 페이지로 이동합니다.

> 🔒 보안을 위해 사용자명은 AES-256으로 암호화되어 저장되며, 비밀번호는 솔트 기반 PBKDF2 해시로 저장됩니다.

### 로그인

가입한 사용자명·비밀번호로 로그인하면 화면 상단에 닉네임과 **로그아웃** 버튼이 표시됩니다.

## 3. 문제 풀기

### 3-1. 문제 목록에서 문제 선택

상단 **문제 목록** 메뉴를 누르면 다음과 같은 표가 나타납니다. 문제마다 **제출 수**, **정답 수**, **정답률**을 한눈에 볼 수 있어 난이도 감을 잡기 좋습니다. 풀고 싶은 문제의 **이름**을 클릭하면 상세 페이지로 이동합니다.

<p align="center">
  <img src="images/1.문제목록.png" alt="문제 목록 화면" width="850">
</p>

> 💡 **정답률**이 100%인 문제는 비교적 쉬운 편이고, 정답률이 0%이거나 제출 수가 적은 문제는 도전적인 편이니 본인 수준에 맞춰 선택해 보세요.

### 3-2. 문제 상세 페이지 구성

문제 페이지는 좌·우 두 영역으로 나뉩니다.

- **좌측**: 문제 정보 (시간/메모리 제한, 지문, 입력 명세, 출력 명세, 예제 입출력)
- **우측**: AI 디미 코딩 선생님 채팅창
- **하단**: 코드 에디터 및 제출 버튼

### 3-3. 코드 작성 & 제출

좌측 하단의 **제출창**에서 다음 순서로 진행합니다.

1. **언어 선택** 드롭다운에서 `Python`, `C`, `C++` 중 하나를 고릅니다.
2. 코드 에디터에 풀이 코드를 작성합니다.
3. **제출** 버튼을 클릭합니다.

<p align="center">
  <img src="images/3.문제제출.png" alt="코드 작성 및 제출 화면" width="850">
</p>

서버에서는 다음 과정이 자동으로 진행됩니다.

1. 격리된 샌드박스 디렉토리 생성
2. 소스 파일 저장 후 컴파일 (C/C++) 또는 구문 검증 (Python)
3. 등록된 테스트케이스를 순차적으로 입력하면서 실행
4. 시간·메모리 제한 내에서 정답 출력과 일치하는지 채점

채점 결과는 즉시 페이지에 반영됩니다.

| 결과 | 의미 |
|---|---|
| **AC** (Accepted) | 모든 테스트케이스 통과 ✅ |
| **WA** (Wrong Answer) | 출력이 정답과 다름 |
| **CE** (Compile Error) | 컴파일 실패 |
| **RE** (Runtime Error) | 실행 도중 오류 |
| **TLE** (Time Limit Exceeded) | 시간 초과 |

## 4. AI 디미 코딩 선생님 활용

문제 페이지 우측의 **AI 디미 코딩 선생님**은 OpenAI GPT 기반의 학습 보조 챗봇입니다. 현재 보고 있는 문제의 지문·입력·출력 명세를 자동으로 파악해 답변하므로, 문제와 관련된 질문을 자유롭게 해보세요.

### 4-1. 기본 질문하기

채팅창 하단 입력란에 질문을 적고 **전송** 버튼(또는 Enter)을 누릅니다.

- 본인이 보낸 메시지는 **오른쪽 파란 말풍선** 💬
- AI 답변은 **왼쪽 회색 말풍선** 🤖

답변에 코드 예시가 포함되면 코드 에디터 박스에 표시되며, 우측 상단의 **Copy** 버튼으로 한 번에 복사할 수 있습니다.

<p align="center">
  <img src="images/2.AI코딩선생님질문.png" alt="AI 코딩 선생님에게 질문하는 화면" width="850">
</p>

### 4-2. 후속 질문 이어가기

대화 맥락은 자동으로 기억됩니다. "방금 설명한 코드를 더 자세히 풀어줘", "다른 방법은 없을까?" 같은 후속 질문으로 단계적으로 학습할 수 있습니다.

<p align="center">
  <img src="images/2.AI코딩선생님질문2.png" alt="AI 코딩 선생님 후속 질문 화면" width="850">
</p>

### 4-3. 효과적인 질문 팁

- ❌ 좋지 않은 예: "정답 알려줘"
- ✅ 좋은 예: "이 문제에서 입력 처리는 어떻게 해야 해?", "내 코드에서 무한루프가 도는데 왜 그런지 힌트만 줘"
- ✅ 좋은 예: "이 자료구조가 왜 효율적인지 설명해줘"

> 💬 **대화 내용 삭제**: 채팅창 상단의 **대화내용 삭제** 버튼을 누르면 현재 문제의 대화 기록이 초기화됩니다. 대화 기록은 브라우저 세션에 저장되므로 같은 문제로 다시 들어와도 대화가 이어집니다.

## 5. 제출 결과 확인

상단 **내 제출** 메뉴에서 본인이 제출한 모든 코드의 결과를 확인할 수 있습니다.

<p align="center">
  <img src="images/4.제출결과확인.png" alt="내 제출 기록 화면" width="850">
</p>

표시되는 정보는 다음과 같습니다.

| 항목 | 설명 |
|---|---|
| **ID** | 제출 번호 (최신 제출이 위) |
| **문제** | 제출한 문제 번호 (클릭 시 문제 페이지로 이동) |
| **언어** | py / cpp / c |
| **결과** | AC (초록) / WA (빨강) / CE / RE / TLE |
| **시간(s)** | 실제 실행 시간 |
| **메모리(MB)** | 사용 메모리 |
| **코드 길이** | 제출한 코드의 문자 수 |
| **코드 보기** | **보기** 링크로 제출 당시의 코드 원본을 다시 확인 |
| **제출 시간** | 정확한 timestamp |

> 📊 **WA가 나왔다면**: 코드 보기 → 예제 입출력 다시 확인 → AI 디미 코딩 선생님에게 힌트 요청 순으로 디버깅해 보세요.

## 6. 직접 문제 출제하기

상단 **문제 제작** 메뉴에서 새 문제를 등록할 수 있습니다.

<p align="center">
  <img src="images/5.문제출제.png" alt="문제 제작 폼" width="850">
</p>

### 입력 항목

| 항목 | 설명 |
|---|---|
| **제목** | 문제의 이름 (예: `(반복문)별 찍기`) |
| **지문 (description)** | 문제 설명 본문. 줄바꿈, 마크다운 일부 지원 |
| **입력 설명 (input specification)** | 입력 형식과 제약 조건 |
| **출력 설명 (output specification)** | 출력 형식 |
| **시간 제한(초)** | 기본 `1`초 |
| **메모리 제한(MB)** | 기본 `128`MB |
| **예제 입력 / 출력** | 사용자가 참고할 수 있도록 보여지는 예시 |
| **체커(선택)** | 출력 비교 로직을 직접 작성하려는 경우의 Python 코드 |

### 테스트케이스 등록

문제 생성 후, 해당 문제의 **테스트케이스 관리** 페이지에서 입력·출력 쌍을 여러 개 추가합니다. 채점은 등록된 모든 테스트케이스를 통과해야 AC로 인정됩니다.

> ⚠️ 예제 입출력만으로는 채점되지 않습니다. 반드시 별도로 **테스트케이스**를 등록해야 합니다.

## 7. 게시판 / 랭킹 / 프로필

### 게시판

상단 **게시판** 메뉴에서 질문/답변을 자유롭게 주고받을 수 있습니다. 본인이 작성한 글만 수정·삭제할 수 있습니다.

### 랭킹

**랭킹** 메뉴는 **해결한 문제 수(distinct)** 기준으로 사용자 순위를 보여줍니다. Codeforces / AtCoder 핸들을 프로필에 연동해 두면 외부 플랫폼의 티어도 함께 표시됩니다.

### 프로필

본인의 닉네임을 클릭하면 프로필 페이지로 이동합니다. 여기서 다음을 확인·수정할 수 있습니다.

- 총 제출 수, 해결한 문제 수, 정답률
- 결과별 통계 (AC / WA / CE / RE / TLE 개수)
- Codeforces / AtCoder 핸들 등록·해제
- 선호 사이트 선택 (랭킹에서 사용할 티어 색상)

## 8. 자주 묻는 질문 (FAQ)

### Q1. 비밀번호를 잊어버렸어요.

비밀번호는 일방향 해시로 저장되어 복구가 불가능합니다. 관리자에게 새 계정 발급을 요청해야 합니다.

### Q2. 채점이 너무 오래 걸려요.

- 문제의 **시간 제한**(보통 1초)을 초과하면 자동으로 TLE로 처리됩니다.
- 알고리즘 자체의 시간 복잡도가 너무 큰 경우가 대부분입니다.
- AI 디미 코딩 선생님에게 "이 풀이의 시간 복잡도는?"이라고 물어보세요.

### Q3. 같은 코드인데 매번 시간이 다르게 나와요.

시스템 부하에 따라 측정값이 미세하게 달라질 수 있습니다. 통과/실패에 영향을 주지 않는 수준의 오차는 정상입니다.

### Q4. "수정 권한이 없습니다" 메시지가 나옵니다.

본인이 작성하지 않은 게시글은 수정/삭제할 수 없습니다. 로그아웃 → 본인 계정으로 다시 로그인했는지 확인해 보세요.

### Q5. AI 디미 코딩 선생님이 응답하지 않아요.

- 인터넷 연결을 확인해 주세요.
- 서버 관리자에게 OpenAI API 키가 정상인지 문의해 주세요.

### Q6. 제출 코드가 잘못 채점된 것 같아요.

테스트케이스가 부족하거나 출력 형식 비교 로직(체커)에 문제가 있을 수 있습니다. 문제 작성자나 관리자에게 게시판으로 문의해 주세요.

## 보안 및 개인정보

- **사용자명**은 AES-256-CBC로 암호화되어 DB에 저장됩니다.
- **비밀번호**는 솔트와 함께 PBKDF2 해시로 저장되어 평문으로 복원할 수 없습니다.
- 세션 쿠키는 `HttpOnly`, `SameSite=Lax`로 설정되어 있어 XSS 공격에 비교적 안전합니다.
- 악성 요청 패턴 차단 및 IP당 Rate Limiting(10초/20회)이 적용되어 있습니다.
